== Overview

Implement the inspection scheduling feature in 4 phases with 10 tasks total. This feature allows BRF administrators to configure and auto-generate inspection schedules for all apartments in a building.

== Phase 1: Data Model and Core Logic

Create the schedule data model interfaces and implement the pure scheduling algorithm.

* [x] TASK-001: Create schedule data model interfaces [parallel]
+
Components:: schedule.ts
Stories:: US-001, US-002, US-003, US-005, US-006, US-007
Specs:: SCHED-001, SCHED-002, SCHED-005, SCHED-006, SCHED-010, SCHED-024, SCHED-029
Description:: Create `src/lib/models/schedule.ts` with TypeScript interfaces: `ScheduleConfig`, `ScheduleAppointment`, `ManualOverride`, `ScheduleResult`. All types as specified in the design.

* [x] TASK-002: Implement schedule validation functions [parallel]
+
Components:: schedule-model.ts
Stories:: US-001, US-002
Specs:: SCHED-003, SCHED-007, SCHED-008
Description:: Create `src/lib/models/schedule-model.ts` with: `createDefaultScheduleConfig()`, `validateDuration()`, `validateDateRange()`, `validateScheduleConfig()`. Include time utility functions: `formatTime()`, `parseTime()`, `isWeekend()`, `addDays()`, `formatDateSwedish()`.

* [x] TASK-003: Implement scheduling algorithm [sequential]
+
Components:: schedule-model.ts
Stories:: US-003, US-004, US-005, US-006, US-007
Specs:: SCHED-011, SCHED-012, SCHED-013, SCHED-016, SCHED-017, SCHED-019, SCHED-020, SCHED-021, SCHED-023, SCHED-027, SCHED-030, SCHED-031
Description:: Implement `getAvailableDates()`, `getMaxAppointmentsPerDay()`, and `calculateSchedule()` in `schedule-model.ts`. The algorithm schedules apartments sequentially floor by floor, respects lunch breaks, excluded dates, weekends, and max per day. Preserves manual overrides. Returns unscheduled count and warning.
