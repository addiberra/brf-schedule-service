== Technical Context

This is an existing project with the building-configuration and inspection-scheduling features already implemented. The message templates feature builds on both -- it uses building data (apartments, floors) and schedule data (appointment dates, times) to resolve template placeholders into personalized messages per apartment.

The project is a Svelte 5 + Vite 6 + TypeScript static site with zero runtime dependencies. All persistence uses browser localStorage with the `brf-schedule:` namespace.

== Decisions and Constraints

* <<../../adrs/0001-svelte5-vite6-typescript-static-site.adoc#,ADR-0001>>: Svelte 5 with runes, Vite 6, TypeScript, Vitest.
* <<../../adrs/0002-browser-localstorage-persistence.adoc#,ADR-0002>>: Browser localStorage with `brf-schedule:*` namespace.

From `docs/constraints.adoc`:

* Zero runtime dependencies
* Static site deployable to GitHub Pages
* All user-facing text in Swedish
* TypeScript strict mode

No new ADRs are needed. The template engine is a simple string replacement using built-in JavaScript string methods -- no library required. Locale-aware date formatting uses the browser-native `Intl.DateTimeFormat` API (no library) with `undefined` locale (browser default) and Swedish as the hardcoded fallback.

== Research and Unknowns

No unknowns remain. The template engine is a straightforward find-and-replace of `{placeholder_name}` tokens. Available data fields are derived from the existing `Apartment` and `ScheduleAppointment` interfaces. The `Intl.DateTimeFormat` API is available in all modern browsers (Chrome 24+, Firefox 29+, Safari 10+) and requires no polyfill.

== System Design

The feature consists of four layers: a data model (TypeScript interfaces), a template engine (pure functions for placeholder parsing and rendering), a persistence extension (storage service), and Svelte UI components.

=== Data Model

All types are defined in `src/lib/models/template.ts`.

[source,typescript]
----
/** A mapping from a placeholder name to a data field */
interface PlaceholderMapping {
  /** User-defined placeholder name (e.g., "lagenhet") */
  name: string;
  /** The data field key this placeholder maps to */
  field: DataField;
}

/** Available data fields that placeholders can map to */
type DataField =
  | 'apartmentId'
  | 'floor'
  | 'position'
  | 'date'
  | 'time'
  | 'dateSwedish'
  | 'dateFormatted';

/** A complete message template */
interface MessageTemplate {
  /** Unique identifier (UUID-like, generated via crypto.randomUUID()) */
  id: string;
  /** User-given template name */
  name: string;
  /** Template body text with {placeholder} tokens */
  body: string;
  /** Placeholder-to-field mappings */
  placeholders: PlaceholderMapping[];
}
----

The `DataField` type enumerates the available fields:

* `apartmentId` -- apartment identifier string (e.g., "1001")
* `floor` -- floor number as string (e.g., "1")
* `position` -- position on floor as string (e.g., "2")
* `date` -- scheduled date in YYYY-MM-DD format (e.g., "2026-03-10")
* `time` -- scheduled start time as HH:MM (e.g., "09:00")
* `dateSwedish` -- Swedish formatted date (e.g., "tisdag 10 mars 2026"). Retained for backward compatibility with saved templates.
* `dateFormatted` -- locale-formatted date using the browser's `Intl.DateTimeFormat` API (e.g., "Thursday, March 10, 2026" for en-US, "torsdag 10 mars 2026" for sv-SE). Falls back to Swedish formatting if locale detection fails.

=== Business Logic: `template-model.ts`

Located at `src/lib/models/template-model.ts`. Pure functions with no side effects.

* `createDefaultTemplate(): MessageTemplate` -- returns a new template with a generated ID, empty name, empty body, and default placeholders.
* `getDefaultPlaceholders(): PlaceholderMapping[]` -- returns suggested placeholders pre-mapped to data fields: `{lagenhet}` → `apartmentId`, `{vaning}` → `floor`, `{datum}` → `dateSwedish`, `{tid}` → `time`.
* `parsePlaceholders(body: string): string[]` -- extracts all `{...}` placeholder names from a template body string. Returns an array of unique names found.
* `resolveDataField(field: DataField, apartment: Apartment, appointment: ScheduleAppointment | null): string` -- resolves a single data field to its string value given an apartment and optional appointment. Returns empty string if appointment is null and a schedule field is requested. For `dateFormatted`, uses `formatDateLocale()` from `schedule-model.ts`. For `dateSwedish`, continues to use the existing `formatDateSwedish()` function for backward compatibility.
* `renderTemplate(template: MessageTemplate, apartment: Apartment, appointment: ScheduleAppointment | null): string` -- replaces all `{placeholder_name}` tokens in the template body with resolved values. For each placeholder found in the body, looks up the mapping in `template.placeholders`. If mapped, resolves the data field. If unmapped, leaves the placeholder as-is (the UI will display unmapped ones in error style).
* `getUnmappedPlaceholders(template: MessageTemplate): string[]` -- returns placeholder names found in the body that have no mapping in `template.placeholders`.
* `generateTemplateId(): string` -- generates a unique ID using `crypto.randomUUID()`.
* `getAvailableDataFields(): { field: DataField; label: string }[]` -- returns the list of all data fields with Swedish display labels for the UI. Includes the new `dateFormatted` field with label "Datum (formaterat)" and retains `dateSwedish` with label "Datum (svenska)" for backward compatibility.

=== Persistence: `storage.ts` (extended)

The existing `storage.ts` is extended with template-specific functions:

* `const TEMPLATES_STORAGE_KEY = "brf-schedule:templates"`
* `saveTemplates(templates: MessageTemplate[]): void` -- serializes all templates to JSON, writes to localStorage.
* `loadTemplates(): MessageTemplate[]` -- reads and parses. Returns empty array if missing or parse fails.
* `clearTemplates(): void` -- removes the key.

=== UI Components

==== `App.svelte` (modified)

Updated to render `MessageTemplatePanel` alongside the building and schedule panels. Passes apartments and schedule result down so templates can resolve placeholders.

==== `MessageTemplatePanel.svelte`

The main orchestrating component for message templates.

State management (Svelte 5 runes):

* `let templates = $state<MessageTemplate[]>(...)` -- initialized from `loadTemplates()`.
* `let selectedTemplateId = $state<string | null>(null)` -- the currently selected template.
* `let editingTemplate = $derived(...)` -- computed from `templates` and `selectedTemplateId` via `templates.find()`. This is a `$derived` value, not `$state`, ensuring it always reflects the current templates array.
* `let previewApartmentId = $state<string | null>(null)` -- the apartment selected for preview.
* Auto-save `$effect` -- persists templates to localStorage and notifies parent when templates change. The save effect must use `$effect` with `untrack` around the parent callback to prevent the callback from becoming a reactive dependency that could interfere with template selection state.

**Template creation flow (TMPL-024):** When the user clicks "Ny mall":

1. `handleNewTemplate()` creates a new template via `createDefaultTemplate()`, appends it to `templates`, and sets `selectedTemplateId` to the new template's ID.
2. The `editingTemplate` derived value resolves to the new template.
3. The `{#if editingTemplate}` conditional renders `TemplateEditor` immediately.
4. The save `$effect` fires and persists the updated list asynchronously without interfering with the selection state.

Props:

* `apartments: Apartment[]` -- apartment list from building config.
* `scheduleResult: ScheduleResult` -- calculated schedule for resolving date/time fields.

Behavior:

* Renders `TemplateList` for the list of saved templates.
* Renders `TemplateEditor` for creating/editing a template. The editor appears immediately when a new template is created (TMPL-024).
* Renders `TemplatePreview` for previewing rendered output.
* Provides "Ny mall" (New template) button that calls `createDefaultTemplate()`.
* Handles save, delete (with `confirm()` dialog), and select operations.

==== `TemplateList.svelte`

Displays saved templates as a selectable list.

Props:

* `templates: MessageTemplate[]` -- all saved templates.
* `selectedId: string | null` -- currently selected template ID.
* `onselect: (id: string) => void` -- callback when a template is selected.
* `ondelete: (id: string) => void` -- callback when delete is requested.

Behavior:

* Renders each template's name in a list.
* Highlights the selected template.
* Each list item has a delete button.

==== `TemplateEditor.svelte`

Edits a single template's name, body, and placeholder mappings.

Props:

* `template: MessageTemplate` -- the template being edited.
* `onsave: (template: MessageTemplate) => void` -- callback on save.

State synchronization:

The editor maintains local editing state (name, body, placeholders) that is a deep clone of the incoming `template` prop. When the `template` prop changes (user selects a different template), the local state must be re-synced. This uses a tracked `$effect` that watches the template's `id` specifically (not the full object reference) to avoid spurious resets during the same editing session.

Behavior:

* Renders a name input field.
* Renders a textarea for the template body.
* Renders a placeholder management section showing all placeholders with their field mappings.
* Provides UI to add new custom placeholders and map them to data fields via a `<select>` dropdown.
* Provides UI to remove placeholders.
* Renders a save button.

==== `TemplatePreview.svelte`

Shows the rendered template for a selected apartment.

Props:

* `template: MessageTemplate` -- the template to preview.
* `apartments: Apartment[]` -- available apartments.
* `scheduleResult: ScheduleResult` -- for resolving schedule fields.
* `selectedApartmentId: string | null` -- the apartment to preview.
* `onselectapartment: (id: string) => void` -- callback for apartment selection.

Behavior:

* Renders an apartment selector dropdown.
* Calls `renderTemplate()` with the selected apartment and its corresponding appointment.
* Displays unmapped placeholders in error style (red text).
* Displays the rendered text preserving whitespace/newlines.

== Story Resolution

=== US-001: Create Message Template

Components:: MessageTemplatePanel, TemplateEditor, template-model
Criteria::
* Text editor for composing -> TemplateEditor renders a textarea for the template body (TMPL-001)
* Template name input -> TemplateEditor renders a name input field (TMPL-002)
* Persist on save -> MessageTemplatePanel calls `saveTemplates()` on save (TMPL-003)
* Immediate editor display on creation -> `handleNewTemplate()` sets `selectedTemplateId`, `editingTemplate` derived resolves, `{#if editingTemplate}` renders editor immediately (TMPL-024)
* Multiple saved templates -> Templates stored as an array in localStorage (TMPL-004)

=== US-002: Insert Placeholders in Template

Components:: TemplateEditor, template-model
Criteria::
* `{placeholder_name}` syntax -> `parsePlaceholders()` extracts tokens matching `{...}` pattern (TMPL-005)
* Create custom placeholder -> TemplateEditor provides add-placeholder UI with name input (TMPL-006)
* Map to data field -> TemplateEditor provides `<select>` dropdown with data fields from `getAvailableDataFields()` (TMPL-007)
* Display available fields -> `getAvailableDataFields()` returns all fields with Swedish labels (TMPL-008)
* Visually distinct in preview -> TemplatePreview renders resolved values for mapped placeholders and error style for unmapped (TMPL-009)

=== US-003: Edit Existing Template

Components:: MessageTemplatePanel, TemplateList, TemplateEditor, storage
Criteria::
* List of saved templates -> TemplateList renders template names (TMPL-010)
* Load into editor on select -> MessageTemplatePanel sets `editingTemplate` from selected template (TMPL-011)
* Update stored template -> `saveTemplates()` writes updated array to localStorage (TMPL-012)
* Delete action -> TemplateList renders delete button per template (TMPL-013)
* Confirmation before delete -> MessageTemplatePanel calls `confirm()` before removing (TMPL-014)

=== US-004: Preview Rendered Template

Components:: TemplatePreview, template-model
Criteria::
* Preview mode with actual data -> TemplatePreview calls `renderTemplate()` with real apartment/appointment data (TMPL-015)
* Apartment selector -> TemplatePreview renders a `<select>` dropdown of apartments (TMPL-016)
* Replace placeholders -> `renderTemplate()` substitutes all mapped placeholders (TMPL-017)
* Error style for unmapped -> `getUnmappedPlaceholders()` identifies unmapped tokens, displayed in red (TMPL-018)

=== US-005: Default Placeholders

Components:: template-model, MessageTemplatePanel
Criteria::
* Suggested placeholders on new template -> `createDefaultTemplate()` calls `getDefaultPlaceholders()` (TMPL-019)
* Pre-mapped to fields -> Default placeholders include `{lagenhet}` → `apartmentId`, `{vaning}` → `floor`, `{datum}` → `dateSwedish`, `{tid}` → `time` (TMPL-020)
* Remove or modify suggested -> TemplateEditor allows removing and re-mapping any placeholder (TMPL-021)

=== US-006: Save Templates

Components:: storage, MessageTemplatePanel
Criteria::
* Persist on create/modify -> `$effect` in MessageTemplatePanel calls `saveTemplates()` when templates change (TMPL-003)
* Restore on load -> MessageTemplatePanel initialization calls `loadTemplates()` (TMPL-022)
* Empty list when no data -> `loadTemplates()` returns `[]`, MessageTemplatePanel displays empty state (TMPL-023)

=== US-007: Locale-Aware Data Field Formatting

Components:: template-model, schedule-model, TemplateEditor, TemplatePreview
Criteria::
* Locale-formatted date rendering -> `resolveDataField('dateFormatted', ...)` calls `formatDateLocale()` which uses `Intl.DateTimeFormat(undefined, { weekday: 'long', day: 'numeric', month: 'long', year: 'numeric' })` (TMPL-025)
* Locale-formatted date field option -> `getAvailableDataFields()` includes `{ field: 'dateFormatted', label: 'Datum (formaterat)' }` (TMPL-026)
* Backward compatibility with dateSwedish -> `resolveDataField('dateSwedish', ...)` continues to call `formatDateSwedish()`. Templates saved with `dateSwedish` mappings still work (TMPL-027)
* Swedish fallback -> `formatDateLocale()` wraps the `Intl.DateTimeFormat` call in a try-catch; on failure, delegates to `formatDateSwedish()` (TMPL-028)
