== Overview

Implementation of the building configuration feature -- the first feature in the project. Includes project scaffolding (Vite 6 + Svelte 5 + TypeScript + Vitest), data model, persistence layer, business logic, and UI components with SVG facade illustration. 4 phases, 10 tasks.

== Phase 2: Core UI Components

Build the input components for building configuration.

* [x] TASK-006: Implement BuildingConfigPanel component [sequential]
+
Components:: src/lib/components/BuildingConfigPanel.svelte
Stories:: US-001, US-002, US-003, US-005, US-006
Specs:: BCFG-001, BCFG-002, BCFG-003, BCFG-004, BCFG-005, BCFG-006, BCFG-007, BCFG-008, BCFG-009, BCFG-010, BCFG-013, BCFG-019, BCFG-020, BCFG-021, BCFG-022, BCFG-023, BCFG-024, BCFG-025
Description:: Main orchestrating component. Uses `$state` for `config` (initialized from `loadBuilding()` or `createDefaultConfig()`), `$state` for `validationErrors` map, `$derived` for `apartments` and `totalCount`. Uses `$effect` to auto-save valid config to localStorage. Renders floor count input, FloorInputRow per floor, total count, reset button (with `confirm()` dialog), and placeholder for BuildingFacade. All UI text in Swedish.
