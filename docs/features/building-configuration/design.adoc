== Technical Context

The building configuration feature is implemented and deployed. This update adds two enhancements: a uniform apartment count mode that simplifies input for buildings with identical floors, and a visually rich SVG facade illustration that replaces the current minimal rendering.

All existing code, persistence, and validation remain unchanged. The enhancements are additive.

== Decisions and Constraints

* <<../../adrs/0001-svelte5-vite6-typescript-static-site.adoc#,ADR-0001>>: Svelte 5 with runes, Vite 6, TypeScript, Vitest. Five dev dependencies only.
* <<../../adrs/0002-browser-localstorage-persistence.adoc#,ADR-0002>>: Browser localStorage for all persistence, namespaced `brf-schedule:*`.

From `docs/constraints.adoc`:

* Zero runtime dependencies -- all packages are dev dependencies
* Static site deployable to GitHub Pages
* All user-facing text in Swedish
* TypeScript strict mode
* A4 print layout (relevant for future features, not this one)

== Research and Unknowns

No unknowns remain. SVG `<defs>`, `<filter>`, `<pattern>`, and `<linearGradient>` are universally supported in modern browsers. No external libraries are needed.

=== Data Model

All types are defined in `src/lib/models/building.ts`.

[source,typescript]
----
/** Configuration for a single floor */
interface FloorConfig {
  /** 1-based floor number */
  floorNumber: number;
  /** Number of apartments on this floor (1–20) */
  apartmentCount: number;
}

/** Complete building configuration */
interface BuildingConfig {
  /** Number of floors (1–30) */
  floorCount: number;
  /** Per-floor configuration, indexed from 0 (floor 1 is index 0) */
  floors: FloorConfig[];
}

/** An apartment with its computed identifier */
interface Apartment {
  /** Unique identifier: floorNumber * 1000 + position (e.g., 1001) */
  id: string;
  /** 1-based floor number */
  floor: number;
  /** 1-based position on floor */
  position: number;
}

/** Validation result for a single field */
interface ValidationResult {
  valid: boolean;
  error?: string;
}
----

NOTE: The `BuildingConfig` interface is unchanged. Uniform mode is purely a UI concern -- when enabled, the panel writes the same apartment count to every floor in the existing `floors` array. No new fields are needed in the persisted model.

=== Business Logic: `building-model.ts`

Located at `src/lib/models/building-model.ts`. Pure functions with no side effects.

Existing functions (unchanged):

* `createDefaultConfig(): BuildingConfig` -- returns a default building with 1 floor, 2 apartments.
* `validateFloorCount(value: number): ValidationResult` -- validates floor count is integer in [1, 30].
* `validateApartmentCount(value: number): ValidationResult` -- validates apartment count is integer in [1, 20].
* `generateApartmentId(floor: number, position: number): string` -- computes `floor * 1000 + position`, returned as a string.
* `generateAllApartments(config: BuildingConfig): Apartment[]` -- generates all apartment objects for the building.
* `totalApartments(config: BuildingConfig): number` -- sums apartment counts across floors.
* `adjustFloors(config: BuildingConfig, newFloorCount: number): BuildingConfig` -- adds or removes floors when floor count changes, preserving existing floor data. New floors default to 2 apartments.

New function:

* `setUniformApartmentCount(config: BuildingConfig, count: number): BuildingConfig` -- returns a new `BuildingConfig` where every floor has `apartmentCount` set to `count`. Preserves `floorCount` and `floorNumber` values. Used by BuildingConfigPanel when uniform mode is active and the shared input changes.

[source,typescript]
----
export function setUniformApartmentCount(
  config: BuildingConfig,
  count: number
): BuildingConfig {
  return {
    ...config,
    floors: config.floors.map((floor) => ({ ...floor, apartmentCount: count })),
  };
}
----

=== Persistence: `storage.ts`

Located at `src/lib/services/storage.ts`. Unchanged.

* `const STORAGE_KEY = "brf-schedule:building"`
* `saveBuilding(config: BuildingConfig): void` -- serializes to JSON, writes to localStorage.
* `loadBuilding(): BuildingConfig | null` -- reads from localStorage, parses JSON. Returns null if key missing or parse fails.
* `clearBuilding(): void` -- removes the key from localStorage.

Uses `try/catch` around all localStorage operations for robustness (private browsing, quota exceeded).

=== UI Components

==== `App.svelte`

Root component. Renders `BuildingConfigPanel` as the main (and currently only) view. Unchanged.

==== `BuildingConfigPanel.svelte`

The main orchestrating component for building configuration.

State management (Svelte 5 runes):

* `let config = $state<BuildingConfig>(...)` -- initialized from `loadBuilding()` or `createDefaultConfig()`.
* `let validationErrors = $state<Map<string, string>>(new Map())` -- keyed by field identifier (e.g., `"floorCount"`, `"floor-1-apartments"`, `"uniformCount"`).
* `let uniformMode = $state<boolean>(false)` -- whether uniform apartment count mode is active. UI-only state, not persisted.
* `let apartments = $derived(generateAllApartments(config))` -- recalculated reactively.
* `let totalCount = $derived(totalApartments(config))` -- recalculated reactively.
* `$effect(() => { ... })` -- persists valid configuration to localStorage whenever config changes.

New behavior for uniform mode:

* A checkbox labeled "Samma antal lägenheter på alla våningar" is rendered below the floor count input.
* When checked (`uniformMode = true`), the `{#each config.floors}` block rendering `FloorInputRow` components is hidden, and a single input field labeled "Antal lägenheter per våning:" is shown instead.
* The single input's value is derived from `config.floors[0].apartmentCount`.
* On change, the handler validates with `validateApartmentCount()` and calls `config = setUniformApartmentCount(config, value)`.
* When the checkbox is toggled ON, the handler calls `config = setUniformApartmentCount(config, config.floors[0].apartmentCount)` to synchronize all floors to floor 1's count.
* When the checkbox is toggled OFF, no changes are made -- the current per-floor values (which are all equal) are preserved and the per-floor inputs reappear.

Existing behavior for floor count, per-floor input, total count, reset, and auto-save is unchanged.

==== `FloorInputRow.svelte`

Unchanged. Props: `floorNumber`, `apartmentCount`, `error`, `onchange`.

==== `BuildingFacade.svelte`

Fully rewritten to render a detailed, visually rich SVG facade. Props remain unchanged: `config: BuildingConfig`, `apartments: Apartment[]`.

===== Rendering Layers (back to front)

1. **Sky background**: Full-viewport `<rect>` with `skyGradient` fill.

2. **Ground area**: A `<rect>` at the bottom spanning full width, filled with `groundGradient`. Simple ellipses (2-3 bushes) positioned along the ground strip in dark green (#3a6b49).

3. **Building body**: A `<rect>` with `wallGradient` fill, with `filter="url(#buildingShadow)"` applied. Positioned from roof base to ground level.

4. **Floor separators**: Horizontal `<line>` elements between each floor in `#c4a882`, `stroke-width="1"`, `opacity="0.6"`.

5. **Windows per apartment**: Each apartment is rendered as a window group:
   * Outer frame: `<rect>` in `#2c3e50`, 2px wider/taller than inner glass.
   * Inner glass: `<rect>` with `glassGradient` fill.
   * Window cross: Vertical and horizontal `<line>` in `#2c3e50`, `stroke-width="0.5"`.
   * Sill: A small `<rect>` below the window, `#8b7355`, protruding 2px on each side.
   * Header: A small `<rect>` above the window, `#8b7355`, protruding 1px on each side.
   * Apartment label: `<text>` centered below the sill, `font-size="9"`, `#2c3e50`.

6. **Balconies**: On alternating apartments (every other `position` on each floor, offset by 1 between floors for staggering), render:
   * A protruding `<rect>` below the window (balcony platform), `#8b7355`, 4px thick.
   * Vertical railing lines: 3 thin `<line>` elements in `#4a4a4a` above the platform.
   * Condition: `(floorNumber + position) % 2 === 0` determines which apartments get balconies.

7. **Entrance door** (ground floor only): Centered horizontally on the building.
   * Outer doorframe: `<rect>` in `#5c4033`.
   * Two door panels: Two `<rect>` elements side by side in `#7a5c47`.
   * Door handles: Two small `<circle>` elements in `#c4a882`.
   * Replaces the center window(s) on floor 1 -- if floor 1 has an odd apartment count, the middle apartment is replaced by the door; if even, the door occupies the center gap between the two middle apartments. Door width equals approximately 1.5x a window width.

8. **Roof**: A `<polygon>` (triangle) in `#a0522d` spanning the building width.
   * Ridge line: A `<line>` along the top edge of the triangle, `#7a3b1e`, `stroke-width="2"`.
   * Chimney: A `<rect>` on the right quarter of the roof, `#6b4c3b`, extending above the roof slope. A small smoke wisp: 2-3 `<path>` elements using quadratic bezier curves (`Q`), `stroke="#cccccc"`, `fill="none"`, `stroke-width="1.5"`, `opacity="0.5"`.

===== Layout Constants

Updated constants for the enhanced facade:

* `APARTMENT_WIDTH = 70` (increased from 60 for sills/headers)
* `APARTMENT_HEIGHT = 55` (increased from 50 for sills/headers)
* `FLOOR_PADDING = 12`
* `WALL_PADDING = 30` (increased for shadows and ground)
* `ROOF_HEIGHT = 50` (increased for chimney)
* `GROUND_HEIGHT = 35` (new -- vegetation strip)
* `WINDOW_INSET = 8` (padding inside the apartment cell for the window)
* `SILL_HEIGHT = 4`
* `HEADER_HEIGHT = 3`
* `BALCONY_DEPTH = 12`

ViewBox:

* Width = `maxApartments * APARTMENT_WIDTH + 2 * WALL_PADDING`
* Height = `floorCount * (APARTMENT_HEIGHT + FLOOR_PADDING) + ROOF_HEIGHT + GROUND_HEIGHT + WALL_PADDING`

The `<svg>` element retains `role="img"` and `aria-label="Fasadillustration av byggnaden"` for screen reader accessibility.

== Story Resolution

=== US-001: Configure Number of Floors

Components:: BuildingConfigPanel, building-model
Criteria::
* Provides an input field for floor count -> BuildingConfigPanel renders a numeric input for `config.floorCount` (BCFG-001)
* Accepts floor count 1-30 -> `validateFloorCount()` enforces [1, 30] range (BCFG-002)
* Displays validation error for out-of-range -> BuildingConfigPanel shows error from `validationErrors` when `validateFloorCount()` returns invalid (BCFG-003)
* Updates illustration on change -> `$state` reactivity propagates changed config to BuildingFacade (BCFG-004)

=== US-002: Configure Apartments per Floor

Components:: BuildingConfigPanel, FloorInputRow, building-model
Criteria::
* Per-floor input field -> One FloorInputRow rendered per floor (BCFG-005)
* Variable apartments per floor -> Each FloorInputRow binds to its own `floors[i].apartmentCount` (BCFG-006)
* Accepts apartment count 1-20 -> `validateApartmentCount()` enforces [1, 20] range (BCFG-007)
* Per-floor validation error -> FloorInputRow receives and displays the error for its specific floor (BCFG-008)
* Illustration updates on change -> Reactive config propagation to BuildingFacade (BCFG-009)
* Displays total apartments -> `totalCount = $derived(totalApartments(config))` rendered in BuildingConfigPanel (BCFG-010)

=== US-003: Apartment Numbering

Components:: building-model, BuildingConfigPanel, BuildingFacade
Criteria::
* Unique identifier per apartment -> `generateApartmentId(floor, position)` produces unique IDs (BCFG-011)
* Identifier format floor*1000+position -> `generateApartmentId()` computes e.g., "1001", "2003" (BCFG-012)
* Recalculate on config change -> `apartments = $derived(generateAllApartments(config))` recalculates reactively (BCFG-013)

=== US-004: Building Facade Illustration

Components:: BuildingFacade, building-model
Criteria::
* Display front-facade illustration -> BuildingFacade renders inline SVG (BCFG-014)
* Dynamic based on config -> SVG viewBox and elements computed from config props (BCFG-015)
* Each apartment as distinct visual element -> SVG rectangles per apartment (BCFG-016)
* Labels with apartment identifier -> SVG `<text>` elements with apartment ID inside each rectangle (BCFG-017)
* Real-time update -> Svelte reactivity re-renders SVG when config props change (BCFG-018)

=== US-005: Save Building Configuration

Components:: storage, BuildingConfigPanel
Criteria::
* Persist on modification -> `$effect` in BuildingConfigPanel calls `saveBuilding(config)` when config changes (BCFG-019)
* Restore on load -> BuildingConfigPanel initialization calls `loadBuilding()` (BCFG-020)
* Default when no saved data -> `loadBuilding()` returns null, BuildingConfigPanel uses `createDefaultConfig()` (BCFG-021)

=== US-006: Reset Building Configuration

Components:: BuildingConfigPanel, storage
Criteria::
* Reset button present -> BuildingConfigPanel renders a reset button (BCFG-022)
* Confirmation before reset -> Button click triggers `confirm()` dialog (BCFG-023)
* Clear and restore defaults on confirm -> Calls `clearBuilding()` + `createDefaultConfig()` (BCFG-024)
* Cancel preserves configuration -> `confirm()` returns false, no action taken (BCFG-025)

=== US-007: Uniform Apartment Count Per Level

Components:: BuildingConfigPanel, building-model
Criteria::
* Toggle control -> Checkbox "Samma antal lägenheter på alla våningar" in BuildingConfigPanel (BCFG-026)
* Single shared input when enabled -> Per-floor FloorInputRow components hidden; single "Antal lägenheter per våning" input shown (BCFG-027)
* Shared input applies to all floors -> Handler calls `setUniformApartmentCount(config, value)` (BCFG-028)
* Uniform count range 1-20 -> Reuses `validateApartmentCount()` (BCFG-029)
* Validation error -> Error displayed next to shared input (BCFG-030)
* Per-floor inputs restored when disabled -> `uniformMode = false` re-shows FloorInputRow components; floor data unchanged (BCFG-031)
* Enable sets all floors to floor 1 count -> `setUniformApartmentCount(config, config.floors[0].apartmentCount)` on toggle ON (BCFG-032)

=== US-008: Enhanced Building Facade Illustration

Components:: BuildingFacade
Criteria::
* Gradient/textured walls -> `wallGradient` applied to building body rect (BCFG-033)
* Window frames with sills and headers -> Each window has outer frame, inner glass, sill rect, header rect (BCFG-034)
* Detailed roof with ridge and chimney -> Roof polygon + ridge line + chimney rect + smoke wisps (BCFG-035)
* Entrance door on ground floor -> Centered double door with handles, replacing center window(s) (BCFG-036)
* Balconies on alternating apartments -> Platform + railing lines where `(floorNumber + position) % 2 === 0` (BCFG-037)
* Floor separator lines -> Horizontal lines between floors in subtle color (BCFG-038)
* Ground area with vegetation -> Green gradient strip + bush ellipses below building (BCFG-039)
* Drop shadows -> SVG `<filter>` with `feDropShadow` on building body (BCFG-040)
* Real-time update -> Svelte reactivity re-renders all SVG elements on config change (BCFG-041)

=== US-009: Configurable Apartment Numbering Start

Components:: BuildingConfigPanel, building-model, storage, BuildingFacade
Criteria::
* Numbering start input -> BuildingConfigPanel adds a numeric input for start value used by floor 1 apartment 1 (BCFG-042)
* Valid set 1001..1901 step 100 -> `validateApartmentNumberStart()` enforces the accepted values (BCFG-043)
* Validation error and prevent save -> Invalid start values are stored in `validationErrors`; autosave remains gated behind zero validation errors (BCFG-044)
* Recalculate all apartment IDs on change -> `generateAllApartments()` uses numbering start and updates all derived apartment identifiers reactively (BCFG-045)
* Preserve uniqueness -> ID generation uses floor+position offsets from the selected start value to ensure unique IDs in the generated apartment list (BCFG-046)
* Legacy storage default -> `loadBuilding()` migration path sets missing numbering start to 1001 for previously saved configurations (BCFG-047)
