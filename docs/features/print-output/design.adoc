== Technical Context

This feature is implemented in an existing Svelte 5 + TypeScript static SPA and is now being migrated to the Tailwind CSS v4 + Bits UI design system introduced by ADR-0003.

The redesign intent is visual and structural (warm professional UI, shared primitives, cleaner print layout architecture) while preserving functional behavior:

1. Individual apartment letters (one A4 page per apartment)
2. Schedule overview (table grouped by date)
3. Native browser print/PDF flow via `window.print()`

No functional behavior changes are intended by this design update.

== Decisions and Constraints

* <<../../adrs/0001-svelte5-vite6-typescript-static-site.adoc#,ADR-0001>>: Svelte 5 with runes, Vite 6, TypeScript, Vitest.
* <<../../adrs/0002-browser-localstorage-persistence.adoc#,ADR-0002>>: Browser localStorage with `brf-schedule:*` namespace.
* <<../../adrs/0003-tailwindcss-v4-bits-ui-warm-tabbed-redesign.adoc#,ADR-0003>>: Tailwind CSS v4 + Bits UI for consistent warm redesign patterns.

From `docs/constraints.adoc`:

* Zero runtime dependencies
* Static site deployable to GitHub Pages
* All user-facing text in Swedish
* TypeScript strict mode
* Print output targets A4 paper size
* PDF generation relies on browser's native print dialog
* No PDF generation library

No additional ADR is required for this feature update. The design reuses existing architectural decisions and adds no new runtime architecture.

== Research and Unknowns

No blocking unknowns remain.

Research conclusions for migration scope:

* Tailwind v4 arbitrary values are appropriate for exact print sizing constraints (`w-[210mm]`, `min-h-[297mm]`, `p-[15mm]`).
* Print media behavior still requires `@media print` and `@page` rules; utility classes complement but do not replace page context rules.
* Bits UI primitives are applicable to PrintPanel controls (mode radio group, template select, print trigger button pattern) without changing domain behavior.
* Existing print data composition in `print-model.ts` remains valid and is reused unchanged.

== System Design

The feature remains split into three layers: data composition (pure model functions), print UI composition (Svelte components + Bits UI primitives), and print media styling (Tailwind utilities + print rules).

=== PrintPanel Control Design (Bits UI)

`PrintPanel.svelte` uses headless primitives for interaction behavior and Tailwind classes for presentation:

* Mode selector uses Bits UI `RadioGroup` with values `letters` and `overview`.
* Template selection uses Bits UI `Select` for `selectedTemplateId` in letters mode.
* Print trigger uses a shared button primitive pattern in the panel control row, with disabled state bound to `canPrint` and action `window.print()`.

State ownership remains unchanged:

* `printMode` (`letters | overview`)
* `selectedTemplateId` (`string | null`)
* `letterData` and `overviewData` as derived read models

=== Business Logic: `print-model.ts`

Located at `src/lib/models/print-model.ts`. Pure functions with no side effects and no migration-driven logic changes.

* `generateLetterData(apartments: Apartment[], scheduleResult: ScheduleResult, template: MessageTemplate): ApartmentLetterData[]` -- renders one letter payload per apartment.
* `generateScheduleOverviewData(scheduleResult: ScheduleResult): ScheduleOverviewData` -- groups appointments by date and formats overview rows.
* `getAppointmentForApartment(apartmentId: string, scheduleResult: ScheduleResult): ScheduleAppointment | null` -- resolves per-apartment schedule information.

=== UI Components

==== `App.svelte` (modified)

Continues to render `PrintPanel` and pass apartments, schedule result, and templates.

==== `PrintPanel.svelte`

The print orchestration component migrated to Tailwind v4 + Bits UI patterns.

Behavior:

* Renders a Bits-based mode selector: "Individuella brev" and "Schemaöversikt".
* In letters mode, renders a Bits-based template select control.
* Renders either `PrintLetterPage` or `PrintScheduleOverview` based on `printMode`.
* Provides a print action button in the same control strip; click calls `window.print()`.
* Keeps existing warning states (no templates, no apartments, no schedule) and disabled print behavior.
* Uses warm professional visual tokens and spacing via Tailwind classes, with no domain-logic impact.

==== `PrintLetterPage.svelte`

Renders a single apartment letter within the print layout.

Behavior:

* Renders letter content in an A4-sized print surface.
* Uses utility classes with arbitrary values for exact dimensions and margins.
* Applies page breaks between letters except on the last rendered letter.
* Preserves whitespace/newlines from rendered template content.

==== `PrintScheduleOverview.svelte`

Renders the schedule overview as a table organized by date.

Behavior:

* Renders heading and summary with warm redesign typography tokens.
* Renders one table per date group (ISO heading + HH:MM rows).
* Uses print-aware utility classes to avoid row/date group splitting and preserve readability.

=== A4 Sizing and Print Media Behavior

The print layout uses exact A4 geometry and print media handling aligned with project constraints:

* Tailwind arbitrary values for surfaces and spacing:
** `w-[210mm]`
** `min-h-[297mm]`
** `p-[15mm]`
** screen-preview scale wrappers such as `scale-[0.7]` only in screen context
* Print media utilities and selectors hide non-print controls and decorative layout wrappers.
* `@page { size: A4; margin: 15mm; }` remains defined in global print CSS to enforce browser print context.
* Page-break semantics:
** letters use `break-after-page` except final item
** overview uses `break-inside-avoid` on logical groups and table segments
* Browser-native print/PDF remains the only output path (`window.print()`), preserving no-PDF-library constraints.

=== Styling and Token Strategy

The print redesign uses Tailwind v4 tokens and utility composition rather than large component-scoped CSS blocks:

* Warm neutral palette for on-screen controls and preview framing.
* Print output palette remains high-contrast black/near-black text on white background.
* Typography choices prioritize legibility in print (11pt+ equivalent with generous line-height).
* Preview-only visual affordances (shadow/border/background) are removed in print media.

== Story Resolution

=== US-001: Print Individual Apartment Letters

Components:: PrintPanel, PrintLetterPage, print-model, template-model
Criteria::
* One A4 page per apartment with rendered template -> `generateLetterData()` produces one `ApartmentLetterData` per apartment with rendered content; `PrintLetterPage` renders each in an A4 container (PRNT-001)
* Open print dialog for individual letters -> PrintPanel calls `window.print()` when user clicks print button in letters mode (PRNT-002)
* Letter fits single A4 page -> `PrintLetterPage` uses `w-[210mm]`, `min-h-[297mm]`, `p-[15mm]` plus `@page` print rules (PRNT-003)
* Page breaks between letters -> `PrintLetterPage` applies page-break utilities/rules between letters (PRNT-004)

=== US-002: Print Schedule Overview

Components:: PrintPanel, PrintScheduleOverview, print-model
Criteria::
* Schedule overview listing all apartments -> `generateScheduleOverviewData()` creates date groups with all appointments (PRNT-005)
* Overview formatted as table by date -> `PrintScheduleOverview` renders a table per date group (PRNT-006)
* Open print dialog for schedule overview -> PrintPanel calls `window.print()` in overview mode (PRNT-007)
* Overview fits A4 with pagination -> `PrintScheduleOverview` uses A4 sizing with print pagination rules (PRNT-008)

=== US-003: Select Print Mode

Components:: PrintPanel
Criteria::
* Print section with two options -> PrintPanel renders Bits `RadioGroup` options for "Individuella brev" and "Schemaöversikt" (PRNT-009)
* Prepare selected print layout -> PrintPanel conditionally renders `PrintLetterPage` or `PrintScheduleOverview` based on `printMode` state (PRNT-010)
* Preview before printing -> PrintPanel renders print layout preview on screen before user initiates print (PRNT-011)

=== US-004: A4 Page Layout

Components:: PrintPanel, PrintLetterPage, PrintScheduleOverview (print media styles)
Criteria::
* Hide UI during print -> print media classes/selectors hide navigation, control strips, and non-print wrappers (PRNT-012)
* Print-friendly font and size -> print typography tokens/classes keep readable 11pt+ equivalent sizing and line height (PRNT-013)

NOTE: A4 dimensions (210mm x 297mm) and minimum 15mm margins are covered by project constraints.

=== US-005: PDF Export via System Print Dialog

Components:: PrintPanel (print media styles, `window.print()`)
Criteria::
* Print-ready layout for PDF export -> `@media print` and `@page` rules ensure layout correctness when saved as PDF (PRNT-014)
* Preserve formatting in PDF -> A4 arbitrary-value sizing plus print media/page-break rules preserve layout in browser PDF export (PRNT-015)

NOTE: No PDF library constraint is covered by project constraints.

=== US-006: ISO Date/Time in Printed Schedule

Components:: print-model, PrintScheduleOverview, PrintLetterPage, template-model
Criteria::
* Date headings in YYYY-MM-DD -> `generateScheduleOverviewData()` exposes raw ISO `date` and `PrintScheduleOverview` renders that value in date headings (PRNT-016)
* Times in HH:MM -> Overview rows continue to use `formatTime()` output and render 24-hour values (PRNT-017)
* Letter output follows template field formatting -> Letter content is still produced by `renderTemplate()` so selected template mappings control date/time style in printed letters (PRNT-018)
