== Overview

This feature adds print output capability to the BRF inspection schedule application. It composes data from building configuration, inspection scheduling, and message templates to produce two printable documents: individual apartment letters and a schedule overview. Historical implementation phases are preserved, and this version adds redesign migration tasks aligned to design v1.2.0 (Tailwind v4 + Bits UI + print layout migration). 5 phases, 16 tasks total.

== Phase 1: Data Model and Business Logic

Define the print data model and implement the pure data composition functions.

* [x] TASK-002: Implement print data composition functions [parallel]
+
Components:: print-model.ts, template-model.ts, schedule-model.ts
Stories:: US-001, US-002
Specs:: PRNT-001, PRNT-005, PRNT-006
Description:: Create `src/lib/models/print-model.ts` with three pure functions: `generateLetterData(apartments, scheduleResult, template)` that renders the template for each apartment and returns `ApartmentLetterData[]`; `generateScheduleOverviewData(scheduleResult)` that groups appointments by date and formats times; and `getAppointmentForApartment(apartmentId, scheduleResult)` that looks up an apartment's appointment.
