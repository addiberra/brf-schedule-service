== Overview

This feature adds print output capability to the BRF inspection schedule application. It composes data from building configuration, inspection scheduling, and message templates to produce two printable documents: individual apartment letters and a schedule overview. Phases 1-3 are implemented. Phase 4 adds ISO date/time presentation in print output. 4 phases, 11 tasks total.

== Phase 1: Data Model and Business Logic

Define the print data model and implement the pure data composition functions.

* [x] TASK-001: Create print data model interfaces [parallel]
+
Components:: print.ts
Stories:: US-001, US-002
Specs:: PRNT-001, PRNT-005
Description:: Create `src/lib/models/print.ts` with TypeScript interfaces: `ApartmentLetterData`, `ScheduleOverviewRow`, `ScheduleDateGroup`, `ScheduleOverviewData`, and `PrintMode` type. These define the data structures for both print modes.

* [x] TASK-002: Implement print data composition functions [parallel]
+
Components:: print-model.ts, template-model.ts, schedule-model.ts
Stories:: US-001, US-002
Specs:: PRNT-001, PRNT-005, PRNT-006
Description:: Create `src/lib/models/print-model.ts` with three pure functions: `generateLetterData(apartments, scheduleResult, template)` that renders the template for each apartment and returns `ApartmentLetterData[]`; `generateScheduleOverviewData(scheduleResult)` that groups appointments by date and formats times; and `getAppointmentForApartment(apartmentId, scheduleResult)` that looks up an apartment's appointment. Uses `renderTemplate()` and `formatTime()` from existing modules.

Stories:: US-001, US-004

* [x] TASK-005: Create PrintPanel component [sequential]

Components:: PrintPanel.svelte, print-model.ts
